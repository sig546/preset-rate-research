# 数据与表述一致性检核清单

> 用途：每次数据更新（`predictions.json` / `actuals.json` / `bond_yields.json` 变更）后、每次发版前，
> 用本清单逐项核查站点对外页面与数据文件的一致性。
> 适用范围：`index.html`、`formula_report.html`、`trigger_analysis_report.html` 及 `data/*.json`。
> 原则：**单一数据源（SSOT）**——所有展示值最终都来自 `data/predictions.json` 与 `data/actuals.json`，
> 页面不得写死、不得与源文件分叉。

---

## 一、数据一致性检查

### 1.1 跨页面 / 跨章节数值一致
- [ ] **C1 预测值唯一来源**：所有页面预测值必须来自 `data/predictions.json`（SSOT），不得写死数值。
  - 检查：`grep -l "predictions.json" *.html` 应为 `index.html`、`trigger_analysis_report.html`；
    页面渲染值应等于 `predictions.json.predictions[].predicted_value`。
  - 位置：`index.html`、`trigger_analysis_report.html`
  - 通过：两页展示的 2026Q2~2027Q1 四值与源文件逐一相等（当前 1.9313 / 1.9094 / 1.8606 / 1.8135）。
  - 项目示例：曾因 `trigger_analysis_report.html` 写死 `1.9383` 与主页 `1.8804` 分叉 → 已修复为 SSOT。

- [ ] **C2 实际值三处一致**：已公布季度的"实际研究值"在 `index.html`、`formula_report.html`、`actuals.json` 三处一致。
  - 检查：`actuals.json.actuals[].value` 与两页"实际"行数值比对。
  - 通过：2024Q4~2026Q1 各季实际值三处相等（如 2026Q1 = 1.93）。

- [ ] **C3 最高值口径一致**：触发规则引用的"普通型最高值"在各页一致且带生效日期。
  - 检查：2025Q3 及之前标 2.50%、2025-09-01 起标 2.00%；页面分段标注正确。
  - 通过：无任何页面把 2026 年季度仍标 2.50% 或混用旧值。

### 1.2 数据来源与引用准确
- [ ] **C4 数据源声明与实际吻合**：`formula_report.html`"数据源"章节所述范围 / 条数与真实文件一致。
  - 检查：国债范围应为 6122 条、2002-01-04~2026-07-08；cdb/ieb 为"混合数据源（货币网近 67 天 + 中债 Excel 补齐）"。
  - 通过：文档条数 / 区间与 `bond_yields.json` 真实统计一致（`python -c "import json;d=json.load(open('data/bond_yields.json'));print(len(d['data']))"`）。

- [ ] **C5 引用脚本与链接有效**：页脚 / 正文引用的脚本名、外链 URL 真实存在且可达。
  - 检查：旧 `public_data_calculation.py` 应已全部改为 `scripts/update_predictions.py`；
    LPR 来源 `shibor.net.cn` 应改为 `chinamoney.com.cn`。
  - 通过：无指向不存在文件或死链的引用。

- [ ] **C6 数据更新时间可见且准确**：消费 `predictions.json` 的页面显示 `last_updated`，且等于文件值。
  - 检查：`trigger_analysis_report.html` 页脚"数据更新时间"应等于 `predictions.json.last_updated`（当前 2026-07-10 13:51）。
  - 通过：页面显示时间 = 文件 `last_updated`，无"报告生成时间"与数据脱钩。

### 1.3 计算结果与原始数据吻合
- [ ] **C7 预测值可复算**：用 `bond_yields.json` + `lpr_history.json` + `deposit_rates.json` 按四步公式离线重算 == `predictions.json`。
  - 检查：以 `scripts/update_predictions.py` 主流程离线复刻（禁用网络抓取）复算四季度。
  - 通过：四值与文件误差 ≤ 0.0001。

- [ ] **C8 分段累加口径正确**：`调整前利率 > 2.0%` 时，`apply_segment` 必须用"分段累加"而非"整值乘系数"。
  - 检查：脚本 `apply_segment` 逐段 `(high-low)*coeff` 累加；关联回溯验证模型 B（avg|差| = 1.42BP）。
  - 通过：历史 6 季度回溯与协会公布值差 ≤ 5BP（模型 B 优于整值乘系数模型 A）。

- [ ] **C9 平推假设成立**：未来债券 / LPR / 定存维持当前水平（不预测下行）。
  - 检查：`extend_bond_to_date` 平推填充未来工作日；`get_lpr_for_month` / `get_deposit_for_month` 对未来返回历史最后值。
  - 通过：未来季度差值均 < 25BP（不触发），序列渐变来自 MA 窗口效应而非利率下行预测。

### 1.4 汇总数据与明细数据对应
- [ ] **C10 回溯汇总 vs 明细**：`formula_report.html` 6 季度汇总数字 == `data/backtest_history.json` 明细。
  - 检查：avg|差| = 1.42BP、max = 2.69BP（2025Q1）、min = 0.42BP（2024Q4）与明细表逐项一致。
  - 通过：摘要数字与明细表无出入。

- [ ] **C11 触发分析主表 vs 明细表**：`trigger_analysis_report.html` 主表（核心指标）与预测明细表同季度数值对应。
  - 检查：同一季度的研究值 / 差值在两表一致。
  - 通过：无同一季度两表数值矛盾。

## 二、表述一致性检查

### 2.1 术语统一
- [ ] **P1 核心术语唯一**："税收溢价"应全篇统一，不得混用"信用利差 / 信用债利差 / 静态利差"。
  - 检查：`grep -rn "信用利差\|信用债利差\|静态利差" *.html` 除刻意的"不再使用静态利差"说明外应无残留。
  - 通过：正文、公式、图表标签均用"税收溢价"。

- [ ] **P2 分段系数表术语一致**：系数区间与口径在文档与脚本一致（如 2.5~3.0 段 = 0.95）。
  - 检查：`formula_report.html` 第五步系数表 vs `scripts/update_predictions.py` 的 `SEGMENT_COEFF`。
  - 通过：两处系数表逐段相等。

### 2.2 同一概念表述唯一
- [ ] **P3 "基础回报"公式表述一致**：`index.html` 与 `formula_report.html` 的基础回报公式文字一致。
  - 检查：均为"基础回报 = MIN【(10Y 国债到期收益率 + 税收溢价) 250 日移动平均, (…+税收溢价) 750 日移动平均】"。
  - 通过：两页公式文字逐字一致。

- [ ] **P4 触发阈值表述一致**：25BP 触发线、连续 2 季度规则在各页一致。
  - 检查：`trigger_analysis_report.html`"紧急调整机制"框 vs `formula_report.html` 触发规则。
  - 通过：无"20BP / 连续 3 季度"等异版表述。

### 2.3 前后逻辑无矛盾
- [ ] **P5 预测趋势逻辑自洽**：正文说明"渐变来自 MA 窗口效应、非预测利率下行"与图表下行趋势不冲突。
  - 检查：图表下行 + 文字解释一致；无"预测利率将下行"之类与平推假设矛盾的措辞。
  - 通过：文字与图形叙事一致。

- [ ] **P6 时间线无矛盾**：各页"最新数据季度 / 报告生成时间"逻辑一致，不出现"未来已公布"的错位。
  - 检查：`actuals` 止于 2026Q1，`predictions` 起于 2026Q2。
  - 通过：无把预测季度误标为"实际"，也无 2026Q1 之前出现预测值。

### 2.4 标题与正文匹配
- [ ] **P7 章节标题贴切**：`formula_report.html` 各 section 标题与内容匹配（如"税收溢价"节内容确为税收溢价）。
  - 检查：逐节核对标题与正文主题。
  - 通过：无标题与内容错配（如标题说"信用利差"但内容已改税收溢价）。

### 2.5 图表说明与正文一致
- [ ] **P8 图表数值与表格一致**：`trigger_analysis_report.html` 两张 Chart 数据 = 表格数据 = `predictions/actuals.json`。

  - 通过：图、表、源三处一致。

- [ ] **P9 图表标注准确**：触发线 25BP、坐标轴范围、图例与正文描述相符。

## 三、格式一致性检查

### 3.1 单位 / 格式 / 标点统一
- [ ] **F1 百分比统一**：利率一律用"%"，同口径保留位数一致（研究值主表 2 位、明细 4 位需显式分区，不得混用）。
  - 检查：对比 `index.html`（2 位）与 `trigger_analysis_report.html` 明细表（4 位）是否各自内部一致。
  - 通过：同位置同口径位数一致，跨页差异有合理解释（非笔误）。

- [ ] **F2 BP 单位统一**：差值单位统一用"BP"，不混用"个基点 / bp / bps"。
  - 检查：`grep -rn "个基点\|bp\b" *.html`。
  - 通过：全站统一"BP"。

- [ ] **F3 小数位数规则明确**：预测值 / 研究值 / MA 值小数位有明确规则且一致。
  - 检查：`predictions.json` 字段（`predicted_value` 4 位、`ma250_gov` 4 位、`spread_250` 4 位…）。
  - 通过：同类字段位数一致，页面展示与源文件位数对应。

- [ ] **F4 标点 / 连接号统一**：中文语境用全角标点；季度区间连接号（`~` 与 `～`）、日期格式（`YYYY-MM-DD`）统一。
  - 检查：全文日期、区间符号风格。
  - 通过：全站标点风格一致，无半角/全角混用。

### 3.2 编号系统连贯
- [ ] **F5 季度编号连贯**：2024Q4 → 2027Q1 连续无重复遗漏，实际 / 预测分界清晰。

  - 通过：无重复季度、无跳号（如缺 2026Q3）。

- [ ] **F6 章节 / 图编号无重复**：`formula_report.html` 步骤编号（第一~第五步）、表编号连续。
  - 检查：步骤序号、表编号。
  - 通过：编号连续无跳号、无重复。

- [ ] **F7 数据文件字段命名一致**：`predictions.json` 各字段名在页面引用与脚本产出一致。
  - 检查：页面读取 `predicted_value` / `base_return` / `spread_250` 等 == JSON 字段名。
  - 通过：页面读取字段名 == JSON 字段名，无拼写漂移。
